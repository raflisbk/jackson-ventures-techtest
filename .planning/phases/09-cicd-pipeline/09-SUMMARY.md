## What Was Built

### `.github/workflows/code-review.yml`
GitHub Actions workflow triggered on `pull_request` (opened / synchronize / reopened):

| Guard | Implementation |
|-------|---------------|
| CI-draft | `github.event.pull_request.draft == false` |
| CI-fork | `github.event.pull_request.head.repo.full_name == github.repository` |
| CI-permissions | `permissions: pull-requests: write, contents: read` |

Steps: checkout → setup-python 3.11 → `pip install openai httpx` → `python scripts/ai_code_review.py`

### `scripts/ai_code_review.py`
Pure-stdlib (no third-party deps beyond openai) script:
1. Fetches PR unified diff via `GET /repos/{owner}/{repo}/pulls/{number}` with `Accept: application/vnd.github.diff`
2. Truncates diff to 12,000 chars if needed
3. Sends to `gpt-4o-mini` with a structured system prompt (correctness, security, performance, test coverage)
4. Posts result as a PR comment via `POST /repos/{owner}/{repo}/issues/{number}/comments`
5. Exits `0` on missing API key (non-fatal — won't fail the build for repos without the secret)

Comment format:
```
## 🤖 AI Code Review

<OpenAI response>

---
*Generated by AI Code Review using OpenAI gpt-4o-mini*
```

## Setup for repo admins
Add `OPENAI_API_KEY` as a repository secret (Settings → Secrets → Actions).
`GITHUB_TOKEN` is automatically provided by GitHub Actions.
